Управление разработкой системы тестирования

November 20, 2024

Планируется разработка приложения для проведения тестирований. Должен быть реализован функционал проведения тестирований, сбора работ тестируемых, проверки ответов (экспертом или автоматически) и представления результатов.

Время разработки: 6 месяцев

Выбранные технологии: Kotlin + Spring Boot, Vue.js, PostgreSQL, Redis, Keycloak

Roadmap проекта

Этап 1: Анализ и проектирование (1 месяц)

  • Анализ интерфейса участников и функционала админки.
  • Проработка технических требований для очного и дистанционного тестирования.
  • Ознакомление с выбранными технологиями.
  • Построение диаграммы классов, диаграммы потоков данных
  • Проектирование базы данных
  • Проектирование API
  • Приоритизация задач

Этап 2: Разработка минимального функционала (MVP, 2 месяца)

  • Настройка проекта
  • Реализация базового функционала (добавление тестирования дистанционного формата и автоматическая проверка)
  • Написание тестов

Этап 3: Тестирование и доработка (1 месяц)

  • Работа с обратной связью от научного руководителя
  • Настройка тестов для фронтенда

Этап 4: Реализация проверки работ (1 месяц)

  • Добавление более специфических функций (сбор работ оффлайн тестирования (загрузка сканов), ручная проверка экспертом)
  • Вывод статистики по участникам, тестам, группам
  • Экспорт данных в Excel/PDF

Этап 5: Финализация и развертывание (1 месяц)

  • Настройка кэширования
  • Деплой
  • Написание текста работы

Инструменты для контроля и управления проектом

Трекер задач: Trello

Методология: Agile (Scrum) 2-недельные спринты, регулярные созвоны с научным руководителем

Возможные риски

Неполное понимание требований

Вероятность: Высокая

Меры минимизации:

Составление и согласование четкого ТЗ, постоянная коммуникация с заказчиком.

Увеличение сроков разработки

Вероятность: Средняя

Меры минимизации:

Использование методологии Agile с короткими спринтами, регулярный пересмотр сроков и задач.
